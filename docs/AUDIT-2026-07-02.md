# Arc90 — In-Depth Audit · July 2, 2026

Method: 8-dimension audit (security, state/dates, timers/audio, a11y, performance, UX/IA,
PWA/offline, data & compliance). Every finding below was verified against the actual code
(file:line cited). Hypotheses that the code refuted are listed at the bottom for honesty.

Severity: P0 broken/dangerous now · P1 significant defect or risk · P2 robustness gap · P3 polish.

---

## P1 — Fix before relying on the feature

### 1. Offline is broken on first install; SW serves HTML as JS (sw.js:29-40) — ✅ FIXED in v64
`SHELL` precaches `./js/app.js` (no query), but pages request `js/app.js?v=63`.
`caches.match(e.request)` does not ignore search params → offline cache MISS → the catch
falls back to `caches.match('/app')` **for every request type**, returning an HTML document
for JS/CSS/image requests. Result: offline app is broken until a full online session has
warmed the runtime cache; even then, asset failures return HTML.
**Fix applied:** `ignoreSearch: true` on fallback match; HTML fallback only for
`request.mode === 'navigate'`; non-200/206 responses no longer cached.

### 2. Sleep-timer auto-off never fires while the screen is locked (js/app.js SOUND_ENGINE) — ✅ FIXED in v64
The 15/30/60-min auto-off uses `setTimeout`. iOS suspends JS timers when the screen locks,
but the `<audio>` element keeps playing — so a "15 min" sound can play all night (battery,
defeats the feature). The `timeupdate` media event *does* keep firing during lock.
**Fix applied:** `timeupdate` listener checks `endAt` and stops playback even when locked
(immediate stop when `document.hidden`, gentle fade in foreground).

### 3. System notifications can never fire on iPhone PWA (js/app.js:7128, 7146)
`new Notification(...)` — the Notification **constructor is unsupported on iOS**
(home-screen PWAs require `ServiceWorkerRegistration.showNotification` + Web Push,
iOS 16.4+). The try/catch swallows the error, so alarm, task-due, 4h/2h "Hardcore"
reminders fire **only as in-app nudges while the app is open** on iPhones.
**Recommendation:** either (a) implement Web Push via the service worker (requires a push
server), or (b) soften the Hardcore/task-reminder copy to "while Arc90 is open" and add
"Add to Home Screen + keep open" guidance. Current UI copy partially discloses this for the
alarm but not for task reminders or Hardcore mode.

### 4. Paid users would never receive Premium (api/stripe-webhook.js ↔ js/app.js)
The webhook correctly writes entitlements to Supabase, but **nothing in the app ever reads
that table** — `S.premium` is a local boolean only. If live Stripe checkout is enabled,
a customer pays and Premium never unlocks.
**Recommendation:** before flipping Stripe live, add an entitlement check (email-based
lookup via a small `/api/entitlement` endpoint) or use StoreKit in the native build.
Do not launch paid checkout until this loop is closed.

### 5. iOS can silently delete all user data after 7 days unused (whole app) — ✅ MITIGATED in v64
All state lives in localStorage + IndexedDB. Safari evicts both after ~7 days of
non-use for non-installed sites — catastrophic for a 90-day program.
**Fix applied:** `navigator.storage.persist()` requested at boot (granted automatically
for home-screen PWAs). **Remaining recommendation:** a periodic "back up your data"
nudge surfacing `exportData()` (already exists in Profile).

### 6. Privacy policy does not disclose analytics (privacy.html, index.html:20, js/app.js track())
`track()` sends events to **Vercel Analytics** (`window.va`) and the app loads
`/_vercel/insights/script.js`, but privacy.html contains zero mention of analytics
(grep: no "analytics/vercel/insights/telemetry"). The brand promise is "private by
default"; undisclosed analytics contradicts it and is a compliance exposure (GDPR/CCPA).
**Recommendation (needs your sign-off, it's legal copy):** add an analytics disclosure
section to privacy.html, or drop Vercel Analytics entirely.

## P2 — Real quality gaps

### 7. Muted text fails WCAG contrast on all 5 themes (css/styles.css tokens)
`--tx-3` computes to ≈2.2–2.6:1 against its backgrounds (e.g. mono:
`rgba(242,242,242,0.30)` on `#0a0a0a` ≈ 2.6:1; light: `rgba(17,17,17,0.32)` on `#f2f2f2`
≈ 2.2:1). It's used for *meaningful* small text: seg-hints, task due times, timer labels.
WCAG AA needs 4.5:1. **Fix:** raise `--tx-3` alphas to ≈0.55 dark / 0.56 light, or stop
using tx-3 for informational text (reserve it for decoration).

### 8. Zoom is disabled (index.html:5)
`user-scalable=no` fails WCAG 1.4.4 (resize text). Standard for app-like PWAs but still an
accessibility failure; low-vision users cannot zoom. Consider `maximum-scale=5` with
`touch-action: manipulation` to prevent double-tap zoom instead.

### 9. Main backup omits proof photos (js/app.js:6839 exportData)
`exportData()` = `JSON.stringify(S)` — complete for state (tasks/journal/health all
included ✓) but proof **photos live in IndexedDB**, not `S`. Restoring a backup on a new
phone loses every photo silently. A separate premium-gated proof export exists (:756).
**Fix:** include photos (base64) in export or state clearly "photos not included".

### 10. Alarm/reminders stop with screen locked, copy only partially says so
Same iOS timer suspension as #2 affects `__arc90AlarmCheck` (20s interval) and the 30s
reminder loop. The alarm card says "keep app in foreground" ✓ but Hardcore mode and task
reminders promise scheduled nudges they can't deliver locked (see #3).

## P3 — Polish

### 11. Protocol/Vitals views highlight the wrong tab (js/app.js mainTabOf)
`mainTabOf('protocol')` returns `'today'` → when browsing Protocol or Vitals (reached from
the Sleep tab), the **Today** tab shows as active. Highlight Sleep instead (their parent).

### 12. 7-tab bar runs at 7–8px labels
Below any comfortable reading size on small phones. Works, but consider folding
Protocol/Vitals-style secondary views into a "More" pattern before adding an 8th tab.

## Addendum — deep slices completed inline (v67)

### 13. Stripe webhook: non-timing-safe HMAC compare (api/stripe-webhook.js:53) — ✅ FIXED
`if (hmac !== v1)` compared signatures with string inequality (timing side-channel) and
`sig.split(/[,=]/)` assumed exact header ordering, breaking during Stripe secret rotation
(multiple v1 entries). **Fix:** defensive key=value parsing, accept any matching v1,
`crypto.timingSafeEqual` with length guard.

### 14. Day counter loses a day at DST spring-forward (js/app.js dayNumber & 3 more) — ✅ FIXED
`Math.floor(diff / DAY_MS)` under-counts when a 23-hour DST day is in the range —
every user's day number (and Forge day) silently dropped by one after the March clock
change. **Fix:** `Math.round` at all four day-count sites. Regression-tested across the
2026-03-08 US spring-forward: old = day 8 (wrong), new = day 9 (correct).

### 15. XSS sweep of flagged sinks — ✅ VERIFIED CLEAN
Systematic scan of interpolations carrying user data (habit names, task titles, journal,
wins, AI chat, proof notes) traced to their render sinks: `showNudge` escapes (esc at
sink), weekly-review `action` escaped at :5089, tip bodies escaped in `renderTipBody`.
No unescaped user-data sink found.

### 16. Keyboard a11y: no Escape to close, toasts not announced (js/app.js) — ✅ FIXED (v68)
Sheets could not be dismissed with Escape (WCAG 2.1.2) and `showNudge` toasts had no
`aria-live`, so screen readers never announced them. **Fix:** global Escape→close-sheet
handler; nudge now `role="status" aria-live="polite"` with an aria-label on its dismiss.
Also verified onboarding: input listeners update the Continue button's disabled state
directly (no stale-button bug).
